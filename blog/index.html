Usar Lektor, más fácil que freír un huevo.

Escrito por Jose Miguel el 2016-10-15

En este primer post del flamante blog, vamos a comenzar hablando sobre Lektor una herramienta pensada para crear sitios estáticos de una forma simple y cómoda.

Lektor se define a sí mismo como un CMS estático fácil de usar, flexible y completo. Su filosofía queda resumida en la frase.

Getting your ideas implemented is as easy as frying an egg.

Como no podía ser de otra forma esta plataforma esta implementada en Python.

Instalación

Existen varias formas pero la que más me gusta consiste en usar el conocido gestor de paquetes pip.

Para ello lo primero que tenemos que hacer es crear un entorno virtual, activarlo y instalar las dependencias de lektor desde el repositorio de pip.

$ virtualenv venv
$ . venv/bin/activate
$ pip install Lektor

Ya esta instalado todo lo necesario.

Iniciar tu primer blog

Para crear tu primer blog, ubícate en un directorio de trabajo y ejecuta en tu consola:

lektor quickstart

El asistente te hace varias preguntas, como el título del blog, el autor y crea un directorio con el mismo nombre que tu blog. Muévete al nuevo directorio y ejecuta:

lektor server

Con el comando anterior se ejecuta un servidor web y puedes ver tu blog accediendo desde el navegador a la url localhost:5000, ademas puedes acceder a la zona de administración accediendo a la url /admin.

Markdown

Para editar los contenidos se usa el lenguaje de marcado Markdown creado en 2004 por John Gruber, cuenta con una sintaxis bastante simple.

Zona de administración

Desde la zona de administración podemos gestionar de forma cómoda los diferentes contenidos, crear nuevos o editar. Además permite subir ficheros "adjuntos", como imágenes o ficheros, para poder enlazarlos directamente desde las entradas.



Modelos

Uno de las características mas interesantes de este framework es la posibilidad de definir los atributos de nuestras entradas. Por ejemplo podemos establecer que una entrada en el blog tenga obligatoriamente, un título, un autor, fecha de creación, url a la imagen principal y un cuerpo. Así requiere al usuario todos los campos y posteriormente la plantilla html conoce los campos del modelo y puede estructurar el contenido de forma adecuada.

Los modelos se definen en el directorio models y sigue un esquema como el siguiente.

[model]
name = Blog Post
label = {{ this.title }}
hidden = yes

[fields.title]
label = Title
type = string
size = large

[fields.main_image]
type = text

Templates

Son archivos html que definen como se estructura y se visualiza los contenidos, incorpora el motor de plantillas Jinja2 que es uno de los motores más eficientes.

Para acceder a los recursos debes usar la api de Lektor.

Plugins

Existe módulos que permiten añadir algunas funcionalidades extra, como integrar sistema de comentarios, redes sociales, google analytics, colorear código fuente, entre otros muchos.

Te recomiendo que te pases por la sección de plugins de la documentación oficial.

Despliegue

Una vez tienes tu blog, lo interesante es subirlo a un sitio público, para ello lo primero que debes hacer es compilarlo. Usando el comando

lektor project-info --output-path

Esto se encarga de convertir tu sitio lektor en un sitio exclusivamente html, css y js.

Una vez compilado, podemos subir el blog manualmente a un servidor, mediante ftp o similar. Sin embargo una forma mucho más cómoda, consiste en definir el método de despliegue en el fichero lektorproject

Por ejemplo para despegarlo por ftp definimos la siguiente estructura.

[servers.production]
target = ftps://myuser:mypassword@ftp.example.com/var/www/example

o para desplegarlo en github pages definimos lo siguiente.

[servers.ghpages]
target = ghpages://your-user/your-repository

Una vez configurado podemos ejecutar el comando:

lektor build && lektor deploy

O pulsar sobre el botón publicar que aparece en la interfaz de administración.



Ahora que conoces como funciona Lektor no tienes escusa para no crear tu entrada en el blog de Python Granada.
